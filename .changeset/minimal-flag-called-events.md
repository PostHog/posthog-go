Send minimal `$feature_flag_called` events when the server enables it. When the `/flags?v=2` response carries `minimalFlagCalledEvents: true` (remote evaluation) or the local-evaluation definitions payload carries `minimal_flag_called_events: true`, and the evaluated flag has `has_experiment: false`, the event keeps only a strict allowlist of evaluation properties (`$feature_flag`, `$feature_flag_response`, `$feature_flag_has_experiment`, `$feature_flag_id`, `$feature_flag_version`, `$feature_flag_reason`, `$feature_flag_request_id`, `$feature_flag_evaluated_at`, `$feature_flag_error`, `locally_evaluated`, `$groups`, `$process_person_profile`, `$geoip_disable`, `$is_server`, `$session_id`, `$window_id`, `$device_id`, `$lib`, `$lib_version`) plus static system context (`$os`, `$os_version`, `$os_distro`, `$go_version`); everything else — including `Config.DefaultEventProperties` and the snapshot path's `$feature/<key>` and `$feature_flag_payload` — is stripped. Any missing signal (gate absent, `has_experiment` unknown, experiment-linked flag, legacy response shapes) keeps today's full event shape. The gate is server-controlled per project; no SDK configuration is added.
